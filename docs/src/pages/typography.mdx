**Polyms Core UI** ships a heading scale as **CSS classes** (`h1`–`h6`) with **Tailwind CSS 4** for size, weight, and alignment utilities. Prefer **semantic** `<h1>`–`<h6>` in markup and attach the matching class when you want this scale—for example `<h2 className='h2'>Section</h2>`. Muted chrome text should use the **slate** ramp (`text-slate-600`, …), not `neutral-*`, so it stays aligned with the rest of the system ([Design System](/design-system)).

## Headings

Classes **`h1`** through **`h6`** set font size, line height, and weight. **`h1`** and **`h2`** step up once from **`lg`** viewport width (`min-width: 64rem`); smaller breakpoints use the compact sizes below.

## Body text & Tailwind utilities

Body copy in your app typically follows **Tailwind** defaults (for example **`text-base`** with comfortable **`leading-*`**). The table below lists common **font-size** and **font-weight** utilities available after you import **`@polyms/core-ui`** styles — use them alongside components from the package.

<Alert variant='info' className='mb-4'>
  **Readable body:** A common default is **`text-base`** (1rem) with line height from **`leading-normal`** or
  Tailwind’s default for that size—tune per layout, not only font size.
</Alert>

## Colored links

Use the **`link`** base class together with a flavor class. Underline appears on **hover** / **focus**, and **visited** / **active** colors follow the semantic tokens. Full examples live on the [Link](/docs/link) page.

Unlike bare **`text-primary-*`** utilities, **`link-*`** encodes interactive states for anchors.

export const coloredLinks = [
  {
    class: '.link',
    variant: 'link',
    description: 'Default: visited/active use `--color-primary`.',
  },
  {
    class: '.link-primary',
    variant: 'link link-primary',
    description: 'Emphasis: `--color-primary-700`, weight 500; visited/active toward `--color-primary-900`.',
  },
  {
    class: '.link-danger',
    variant: 'link link-danger',
    description: 'Destructive: `--color-danger`; visited/active `--color-danger-700`.',
  },
  {
    class: '.link-light',
    variant: 'link link-light',
    description:
      'Muted `--color-slate-500`; hover `--color-primary-600`; visited/active `--color-slate-900`.',
  },
]

<div className='card mt-2'>
  <table className='table-striped table-borderless table align-middle'>
    <thead className='thead-light'>
      <tr>
        <th className='w-px'>Class</th>

  </table>
</div>

## Stretched link

The **`stretched-link`** class adds a **`::after`** overlay (`position: absolute; inset: 0; z-index: 10`) on the anchor so a **positioned** parent (usually **`relative`**, e.g. **`card relative`**) becomes fully clickable. Details and stacking notes: [Link — Stretched link](/docs/link).

export const stretchedLinkCode = `<div className='card relative max-w-md'>
  <div className='card-body'>
    <h5 className='card-title'>Card with stretched link</h5>
    <p className='card-text'>
      Some quick example text to build on the card title and make up the bulk of the card's content.
    </p>
    <a href='#stretched-link' className='link link-primary stretched-link'>
      Go somewhere
    </a>
  </div>
</div>`

<Alert variant='info' className='mt-8' badge='::: TIP :::'>
  The card (or other wrapper) must include **`relative`** so the overlay positions against it. Raise
  **`z-index`** on controls that must stay above the overlay.
</Alert>
