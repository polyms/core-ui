**Polyms Core UI** ships a bundled **agent skill** so AI coding assistants (Cursor, Claude Code, and other `AGENTS.md`-aware tools) follow the same composition rules you do: barrel imports, slate grays, semantic tones, compound component trees, programmatic overlays, and the CSS-only utilities. Instead of pasting conventions into every prompt, install the skill once into your repository.

## What the skill contains

The skill is split for progressive disclosure — agents load only the part relevant to the task. **`SKILL.md`** carries `name: core-ui` (skill identifier). Reference files (`field.md`, `modal.md`, …) use **`description`** frontmatter only — not a second `name`, per [Cursor skill conventions](https://cursor.com/docs/context/skills).

- **SKILL.md** — entry: **skill routing** (keyword → file), hard rules, workflow.
- **components.md** — `Button`, Toast, compound trees (Menu, NavigationMenu, Toolbar, Select, Tabs, Checkbox, Radio, Switch, Popover, Tooltip, Avatar, Breadcrumb, Accordion, Alert, NumberField, Collapsible, Spinner, `dynamic`), `useRender`; Field → **field.md**; Modal & Offcanvas → **modal.md**.
- **theming.md** — slate vs neutral, semantic tones, dark mode tokens, logical-direction spacing.
- **css-utilities.md** — table, badge, card, link, and the `item-*` interaction recipe.
- **setup.md** — install, styles, app shell, Toast; programmatic wiring points to **modal.md#programmatic-overlays**.
- **quality.md** — task inference, anti-slop, copy tells, visual consistency (shape/typography), list primitive choice, output completeness (no truncation), field label copy, pre-flight checks.
- **redesign.md** — migrate existing app screens to `@polyms/core-ui` (Scan → Diagnose → Fix, preserve vs overhaul, audit checklist, redesign pre-flight).
- **brandkit.md** — map brand identity to `primary-*` and semantic tokens, component touchpoints, optional `BRAND.md`, brand pre-flight (code implementation, not image generation).
- **field.md** — `Field` vs `Field.Floating`, child order, icons, validation, anti-patterns.
- **modal.md** — Modal & Offcanvas (same overlay pattern): trees, programmatic `showModal` / `showOffcanvas`, anti-patterns. Compose with **field.md** when body has forms.

## Install the skill

After adding the package, run the bundled installer from your repository root:

```bash
pnpm exec core-ui-skill
```

```bash
npx core-ui-skill
```

The installer copies the skill into the discovery folders that agents read:

- `.cursor/skills/core-ui` — Cursor
- `.claude/skills/core-ui` — Claude Code
- `.agents/skills/core-ui` — generic `AGENTS.md` tools

## Options

```bash
core-ui-skill [options]
```

- **`--target <all|cursor|claude|agents>`** — install one target instead of all (default: `all`).
- **`--cwd <path>`** — repository root to install into (default: current directory).
- **`--force`** — overwrite an existing installed skill.
- **`--dry-run`** — print planned changes without writing files.
- **`-h`, `--help`** — show usage.

Install for a single tool:

```bash
core-ui-skill --target cursor
```

Preview without writing:

```bash
core-ui-skill --dry-run
```

## Keep it in sync

The skill is versioned with the package, so re-run the installer after upgrading `@polyms/core-ui`. Add a script so the whole team syncs the same way:

```json
{
  "scripts": {
    "core-ui:skill": "core-ui-skill --force"
  }
}
```

```bash
pnpm core-ui:skill
```

## Verify

After installing, your agent should automatically apply the rules when you work with `@polyms/core-ui`. Confirm the files exist:

```bash
ls .cursor/skills/core-ui
```

If a rule ever disagrees with the TypeScript types for your installed version, trust **`node_modules/@polyms/core-ui/index.d.ts`**.

## Brand kit in your repo

For product branding (accent color, surfaces, component mapping), add two files in the consumer app:

1. **`src/styles/theme.css`** — import `@polyms/core-ui/styles/tailwind.css`, then override `--color-primary-*` and optional surface tokens. See [Design System — Brand theming](/design-system#brand-theming).
2. **`BRAND.md`** (repo root or `docs/`) — strategy, token hex values, and which components express the brand. Agents read this when **`brandkit.md`** is installed via the skill.

Prompt example for Cursor or Claude Code:

```text
Follow core-ui brandkit.md. Our BRAND.md is at ./BRAND.md. Update theme.css primary ramp to match.
```

The skill file **`brandkit.md`** ships inside `.cursor/skills/core-ui/` (and sibling paths) after install — it is the machine-readable counterpart to your human **`BRAND.md`**.
