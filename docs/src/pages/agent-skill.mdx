After you install **`@polyms/core-ui`**, you can add a bundled **agent skill** so AI coding assistants — Cursor, Claude Code, and other tools that read `AGENTS.md` — compose UI the same way this library expects. Install it once in your repository instead of pasting conventions into every prompt. For the visual and composition rules the skill enforces, see [Design System](/design-system).

## What you get

The skill teaches your assistant how to work with Core UI in common product tasks:

The installer copies the skill into the folders your tools discover automatically:

- `.cursor/skills/core-ui` — Cursor
- `.claude/skills/core-ui` — Claude Code
- `.agents/skills/core-ui` — generic `AGENTS.md` tools

## Install the skill

You need **`@polyms/core-ui`** in the project first — see [Installation](/installation). From your repository root, run the bundled installer:

```bash
pnpm exec core-ui-skill
```

```bash
npx core-ui-skill
```

## Options

```bash
core-ui-skill [options]
```

- **`--target <all|cursor|claude|agents>`** — install for one tool only (default: all).
- **`--cwd <path>`** — repository root to install into (default: current directory).
- **`--force`** — overwrite an existing skill (useful after upgrading the package or re-syncing the team).
- **`--dry-run`** — show what would be written without changing files.
- **`-h`, `--help`** — show usage.

Install for a single tool:

```bash
core-ui-skill --target cursor
```

Preview without writing:

```bash
core-ui-skill --dry-run
```

## Keep it in sync

The skill ships with the package and updates when you upgrade **`@polyms/core-ui`**. Re-run the installer after each upgrade. Add a script so everyone on the team syncs the same way:

```json
{
  "scripts": {
    "core-ui:skill": "core-ui-skill --force"
  }
}
```

```bash
pnpm core-ui:skill
```

## Verify

Confirm the skill is on disk:

```bash
ls .cursor/skills/core-ui
```

Then try a real prompt in your assistant — for example:

```text
Create a login form with Field, a primary submit Button, and inline validation errors.
```

The output should use `@polyms/core-ui` imports and match the patterns in the [Design System](/design-system). If anything disagrees with your installed version, trust **`node_modules/@polyms/core-ui/index.d.ts`**.

## Brand kit in your repo

To keep AI-generated UI on-brand, add a theme file (for example `src/styles/theme.css`) and a short **`BRAND.md`** in your app with accent colors, surfaces, and which components carry the brand. Full setup and examples: [Design System — Brand theming](/design-system#brand-theming).

Example prompt:

```text
Our BRAND.md is at ./BRAND.md. Update theme.css so the primary ramp matches our brand tokens.
```
