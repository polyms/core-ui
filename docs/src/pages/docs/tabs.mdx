Organize and navigate between related content groups without leaving the page. Tabs provide a clean interface for switching between different views while maintaining context and reducing cognitive load.

## Usage

```tsx
import { Tabs } from '@polyms/core-ui'

const tabs = (
  <Tabs defaultValue='profile'>
    <Tabs.List>
      <Tabs.Tab value='profile'>Profile</Tabs.Tab>
      <Tabs.Tab value='settings'>Settings</Tabs.Tab>
      <Tabs.Tab value='notifications'>Notifications</Tabs.Tab>
    </Tabs.List>

    <Tabs.Panel value='profile'>Profile content goes here</Tabs.Panel>
    <Tabs.Panel value='settings'>Settings content goes here</Tabs.Panel>
    <Tabs.Panel value='notifications'>Notifications content goes here</Tabs.Panel>
  </Tabs>
)
```

## Behavior

### Active Indicator

The tabs component includes an animated indicator that smoothly transitions between tabs as users navigate, providing clear visual feedback about the current selection.

### Keyboard Navigation

Tabs support full keyboard navigation for accessibility:

- **Arrow Left/Right** - Navigate between tabs (horizontal orientation)
- **Arrow Up/Down** - Navigate between tabs (vertical orientation)
- **Home** - Move to first tab
- **End** - Move to last tab
- **Tab** - Move focus to/from the tab panel

## Responsive layout

Tab labels do not wrap by default. When you have several peer tabs on a phone-width layout, wrap **`Tabs.List`** in a horizontal scroll container. For settings-style pages with many sections, switch to **`orientation='vertical'`** and let the list become a side rail from your **`md:`** breakpoint.
