A two-state button that can be on or off. Use it for binary settings where the control itself reflects the state — like bold, mute, or favorite — and pair it with an icon for a compact, immediate-feedback action.

## Usage

```tsx
import { Toggle } from '@polyms/core-ui'

const toggle = (
  <Toggle className='toggle' aria-label='Bold'>
    B
  </Toggle>
)
```

Add the **`toggle`** and **`toggle-group`** classes to match the styling in the examples below.

## Controlled vs Uncontrolled

### Uncontrolled

Use `defaultPressed` to set the initial state and let the component manage itself:

```tsx
const toggle = <Toggle className='toggle' defaultPressed aria-label='Favorite' />
```

### Controlled

Drive the pressed state externally with `pressed` and `onPressedChange`:

```tsx
import { Toggle } from '@polyms/core-ui'
import { useState } from 'react'

export default function ControlledToggle() {
  const [pressed, setPressed] = useState(false)

  return <Toggle className='toggle' pressed={pressed} onPressedChange={setPressed} aria-label='Mute' />
}
```

## Toggle group

Wrap related toggles in a `ToggleGroup` to share state. By default the group is exclusive (single selection, like a segmented control); pass `multiple` to allow several toggles to be pressed at once.

```tsx
import { Toggle, ToggleGroup } from '@polyms/core-ui'

const toggleGroup = (
  <ToggleGroup className='toggle-group' aria-label='Alignment' defaultValue={['left']}>
    <Toggle className='toggle' value='left'>
      Left
    </Toggle>
    <Toggle className='toggle' value='center'>
      Center
    </Toggle>
    <Toggle className='toggle' value='right'>
      Right
    </Toggle>
  </ToggleGroup>
)
```

## Composition

`Toggle` composes with other primitives through the `render` prop. Inside a `Toolbar`, pass it to `Toolbar.Button` so it inherits roving keyboard focus:

```tsx
import { Toggle, ToggleGroup, Toolbar } from '@polyms/core-ui'

const toolbar = (
  <Toolbar>
    <ToggleGroup className='toolbar-group' aria-label='Alignment'>
      <Toolbar.Button render={<Toggle />} value='left'>
        Left
      </Toolbar.Button>
      <Toolbar.Button render={<Toggle />} value='right'>
        Right
      </Toolbar.Button>
    </ToggleGroup>
  </Toolbar>
)
```

## Guidelines

- Always provide an `aria-label` (or visible text) so screen readers can announce the toggle's purpose.
- Prefer a Switch when the action takes effect immediately on a setting; use a Toggle when the button itself represents an on/off state (e.g. formatting tools).
- Give every `Toggle` inside a `ToggleGroup` a unique `value`.
- Use `multiple` for independent options and the default exclusive mode for mutually exclusive choices.
