Navigation Menu is a multi-item navigation surface designed for site headers and **mega menus**. Triggers share a single popup that smoothly animates between item sizes and slides content based on the activation direction. Built on top of [Base UI Navigation Menu](https://base-ui.com/react/components/navigation-menu).

The example above composes a **featured gradient card**, multi-column link sections with **per-item badges**, a quoted testimonial with a stats panel, and an inline **footer CTA strip** — all sitting inside the single shared `NavigationMenu.Viewport`, so transitions between items stay smooth. A few patterns worth lifting from it:

- **Featured tile** — wrap a `NavigationMenu.Link` with custom utility classes (`group`, `relative`, gradient `style`) to render a hero card. Auto-close on click still fires because it's a real `Link`.
- **Per-item badges** — use the design-system `badge` + tone classes (`badge-primary`, `badge-success`, …) for status pills next to titles instead of hand-rolled spans.
- **Section labels** — use `NavigationMenu.GroupLabel` above each column. Keep items inside a `<ul>` so each link is a real list item.
- **Footer strip** — use `NavigationMenu.Footer` after the grid for changelog teasers or sales CTAs. It animates with the rest of the popup.
- **Mixed link variants** — pair plain `Link`s for dense rows with `variant='soft'` for discoverable cards; `variant='danger'` for destructive rows; `variant='trigger'` for static *Pricing* / *Login* entries in the top list.

For a **fintech app-shell** variant — `variant='bare'` list inside a sticky page header, with balance cards, quick-action grids, and mini analytics — see the [Demo playground](/demo).

## Usage

```tsx
import { NavigationMenu } from '@polyms/core'

<NavigationMenu>
  <NavigationMenu.List>
    <NavigationMenu.Item value='products'>
      <NavigationMenu.Trigger>
        Products
        <NavigationMenu.Icon />
      </NavigationMenu.Trigger>
      <NavigationMenu.Content>
        <NavigationMenu.Link href='/launchpad'>Launchpad</NavigationMenu.Link>
        <NavigationMenu.Link href='/analytics'>Analytics</NavigationMenu.Link>
      </NavigationMenu.Content>
    </NavigationMenu.Item>
  </NavigationMenu.List>

  <NavigationMenu.Viewport />
</NavigationMenu>
```

Each `Content` is teleported into the single `Viewport` you drop at the end of the root, so transitions between items stay smooth and accessible. `NavigationMenu.Viewport` is the all-in-one Polyms shell — it wires up the portal, positioner, popup, and arrow for you.

## Composition

### App header shell

Use `NavigationMenu.List variant='bare'` when the surrounding app header already owns the nav chrome (container width, sticky background, border, blur, etc.). Pair it with `size='lg'` on `NavigationMenu.Trigger` and trigger-style links for roomier marketing/header navigation.

```tsx
<NavigationMenu delay={100} closeDelay={120}>
  <NavigationMenu.List variant='bare'>
    <NavigationMenu.Item value='about'>
      <NavigationMenu.Trigger size='lg'>
        About
        <NavigationMenu.Icon />
      </NavigationMenu.Trigger>
      <NavigationMenu.Content>{/* mega menu */}</NavigationMenu.Content>
    </NavigationMenu.Item>

    <NavigationMenu.Item>
      <NavigationMenu.Link href='/contact-us' size='lg' variant='trigger'>
        Contact
      </NavigationMenu.Link>
    </NavigationMenu.Item>
  </NavigationMenu.List>

  <NavigationMenu.Viewport arrow={false} />
</NavigationMenu>
```

### With a static link

Render a `NavigationMenu.Link` directly inside `NavigationMenu.Item` (without a trigger/content pair) for a plain navigation entry — perfect for *Pricing*, *Login*, etc. Pass **`variant='trigger'`** so it inherits the same pill padding, radius, hover, and animated underline as `NavigationMenu.Trigger` and stays perfectly aligned inside the list.

```tsx
<NavigationMenu.List>
  <NavigationMenu.Item value='products'>
    <NavigationMenu.Trigger>
      Products
      <NavigationMenu.Icon />
    </NavigationMenu.Trigger>
    <NavigationMenu.Content>{/* mega menu */}</NavigationMenu.Content>
  </NavigationMenu.Item>

  <NavigationMenu.Item>
    <NavigationMenu.Link href='/pricing' variant='trigger'>
      Pricing
    </NavigationMenu.Link>
  </NavigationMenu.Item>
</NavigationMenu.List>
```

### Link variants

By default, `NavigationMenu.Link` renders a plain row with a slate hover background. Use variants when a link needs stronger semantics: **`soft`** for discoverable card rows, **`danger`** for destructive actions, and **`trigger`** for static top-level entries.

```tsx
<NavigationMenu.Content>
  <NavigationMenu.Link href='/docs'>Documentation</NavigationMenu.Link>

  <NavigationMenu.Link href='/community' variant='soft'>
    Community
  </NavigationMenu.Link>

  <NavigationMenu.Link active href='/showcase' variant='soft'>
    Showcase
  </NavigationMenu.Link>

  <NavigationMenu.Separator />

  <NavigationMenu.Link href='/revoke-access' variant='danger'>
    Revoke access
  </NavigationMenu.Link>
</NavigationMenu.Content>
```

`soft` reacts to `:hover`, `:focus-visible`, and `data-active` (set automatically by the `active` prop) — border flips to solid, background tints stronger. `danger` keeps the plain-row layout but swaps hover/focus/active states to the danger tone.

### Grouping content

Use `GroupLabel`, `Separator`, and `Footer` to keep mega menu panels consistent without repeating utility bundles:

```tsx
<NavigationMenu.Content>
  <NavigationMenu.GroupLabel>Products</NavigationMenu.GroupLabel>
  <NavigationMenu.Link href='/launchpad'>Launchpad</NavigationMenu.Link>
  <NavigationMenu.Link href='/analytics'>Analytics</NavigationMenu.Link>
  <NavigationMenu.Separator />
  <NavigationMenu.Footer>
    <span>Postgres branching is now GA.</span>
    <a className='link link-primary' href='/changelog'>Read changelog</a>
  </NavigationMenu.Footer>
</NavigationMenu.Content>
```

### Controlled value

Drive the open item externally — useful for guided tours, analytics, or syncing with the URL.

```tsx
import { NavigationMenu } from '@polyms/core'
import { useState } from 'react'

function ControlledNavigation() {
  const [value, setValue] = useState<string | null>(null)

  return (
    <NavigationMenu onValueChange={setValue} value={value}>
      <NavigationMenu.List>
        <NavigationMenu.Item value='products'>
          <NavigationMenu.Trigger>Products</NavigationMenu.Trigger>
          <NavigationMenu.Content>{/* … */}</NavigationMenu.Content>
        </NavigationMenu.Item>
      </NavigationMenu.List>
      <NavigationMenu.Viewport />
    </NavigationMenu>
  )
}
```

### Framework links

Wrap `NavigationMenu.Link` with your router's component via the `render` prop:

```tsx
import { Link as RouterLink } from '@tanstack/react-router'
import { NavigationMenu } from '@polyms/core'

function NavLink({ href, ...props }: NavigationMenu.Link.Props & { href: string }) {
  return <NavigationMenu.Link render={<RouterLink to={href} />} {...props} />
}
```

The `active` prop sets `aria-current='page'` automatically — drive it from your router's match state.

### Nested submenus

Nest a second `NavigationMenu` inside `NavigationMenu.Content` for multi-level mega menus. Nested roots render a `<div>` instead of `<nav>` and share hover coordination with the parent.

```tsx
<NavigationMenu.Content>
  <NavigationMenu>
    <NavigationMenu.List>
      <NavigationMenu.Item value='overview'>
        <NavigationMenu.Trigger>Overview</NavigationMenu.Trigger>
        <NavigationMenu.Content>{/* deeper links */}</NavigationMenu.Content>
      </NavigationMenu.Item>
    </NavigationMenu.List>
    <NavigationMenu.Viewport />
  </NavigationMenu>
</NavigationMenu.Content>
```

## Positioning

Pass positioner props directly to `NavigationMenu.Viewport` — `side`, `align`, `sideOffset`, `collisionPadding`:

```tsx
<NavigationMenu.Viewport align='start' side='bottom' sideOffset={12} />
```

Set `arrow={false}` to hide the pointer:

```tsx
<NavigationMenu.Viewport arrow={false} />
```

## Hover timing

Tune **`delay`** (open) and **`closeDelay`** (close) on the root to balance responsiveness against accidental triggers. Both default to **`50ms`**; raise `delay` to 150–300ms for forgiving hover targets, drop `closeDelay` to 0 for a snappier dismissal.

```tsx
<NavigationMenu closeDelay={0} delay={200}>
  {/* … */}
</NavigationMenu>
```

## Keyboard Navigation

- **Arrow Left / Arrow Right**: Move focus between top-level triggers (horizontal orientation).
- **Arrow Up / Arrow Down**: Move focus between top-level triggers (vertical orientation) or into the open popup.
- **Enter / Space**: Open the focused trigger's popup.
- **Tab**: Move into the open popup; subsequent `Tab` keys cycle through links inside.
- **Escape**: Close the popup and return focus to the active trigger.
