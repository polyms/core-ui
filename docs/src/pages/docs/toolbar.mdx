A container for grouping a related set of buttons, toggles, links, and inputs. Toolbar provides roving keyboard focus, accessible labelling, and seamless composition with other Base UI primitives.

## Usage

```tsx
import { Toolbar } from '@polyms/core-ui'

<Toolbar aria-label='Formatting'>
  <Toolbar.Group aria-label='Text formatting'>
    <Toolbar.Button>Bold</Toolbar.Button>
    <Toolbar.Button>Italic</Toolbar.Button>
  </Toolbar.Group>
  <Toolbar.Separator orientation='vertical' />
  <Toolbar.Link href='#'>Docs</Toolbar.Link>
</Toolbar>
```

## Composition

`Toolbar.Button` is the primary slot for any clickable control. Use the `render` prop to compose it with other primitives so they inherit toolbar keyboard navigation:

```tsx
import { Toggle, ToggleGroup, Toolbar } from '@polyms/core-ui'

<Toolbar>
  <ToggleGroup className='toolbar-group' aria-label='Alignment'>
    <Toolbar.Button render={<Toggle />} value='left'>Left</Toolbar.Button>
    <Toolbar.Button render={<Toggle />} value='right'>Right</Toolbar.Button>
  </ToggleGroup>
</Toolbar>
```

The same pattern applies to `Menu.Trigger`, `Select.Trigger`, `Popover.Trigger`, `Dialog.Trigger`, and `AlertDialog.Trigger`.

You can also render the `Button` component to reuse its icon sizing and tooltip:

```tsx
import { Button, Toolbar } from '@polyms/core-ui'

<Toolbar>
  <Toolbar.Button render={<Button icon tooltip='Bold' />}>
    <HugeiconsIcon icon={TextBoldIcon} size={16} strokeWidth={2} />
  </Toolbar.Button>
</Toolbar>
```

The toolbar owns the visual shell, so `.toolbar-button` wins over `.btn` for geometry and the ghost palette — `Button` `variant`/`outlined` colors are intentionally flattened to keep toolbar items uniform. Horizontal padding is reconciled automatically, so `icon` buttons stay square. Use `Button` here for its `icon`, `tooltip`, and `render`-friendly API rather than for a colored variant.

For tooltips, do the opposite — pass the toolbar item as the `render` of `Tooltip.Trigger`:

```tsx
<Toolbar>
  <Tooltip>
    <Tooltip.Trigger render={<Toolbar.Button />}>Bold</Tooltip.Trigger>
    <Tooltip.Content>Toggle bold</Tooltip.Content>
  </Tooltip>
</Toolbar>
```

## Vertical orientation

Pass `orientation='vertical'` to switch keyboard navigation to the up/down keys and stack items vertically. Use a horizontal `Toolbar.Separator` inside a vertical toolbar.

## Behavior

- **Roving focus** — Only one toolbar item is in the tab order at a time.
- **Keyboard navigation** — Arrow keys move focus between items, `Home`/`End` jump to the edges; focus loops by default (set `loopFocus={false}` to disable).
- **Disabled items** — Stay focusable by default so screen readers can announce them. Set `focusableWhenDisabled={false}` to skip them while navigating.
- **Inputs** — Place `Toolbar.Input` last in a horizontal toolbar so the left/right keys keep working as a text-cursor inside the input.

## Guidelines

- Always provide `aria-label` (or `aria-labelledby`) on `Toolbar` and on each toolbar group so screen readers can announce purpose.
- Group related controls with `Toolbar.Group` (or `ToggleGroup` when you need exclusive selection).
- Use one `Toolbar.Input` per horizontal toolbar and place it at the end of the row.
