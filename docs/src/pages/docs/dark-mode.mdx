Class-based dark mode for **`@polyms/core-ui`**. Add **`.dark`** to `<html>` (or any wrapper) and semantic utilities switch between light and dark palettes at runtime — no CSS rebuild.

Dark body background defaults to **`#2e3440`** (Nord Polar Night). Tokens live in **`core/src/styles/_root.css`**; Tailwind v4 **`@theme inline`** keeps **`var()`** resolution at runtime. Built-in components use these tokens and auto-invert when the theme toggles.

## Setup

Enable dark mode by adding the class to the document root (or a subtree):

```html
<html class="dark">
```

Optional sync with system preference:

```tsx
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
document.documentElement.classList.toggle('dark', prefersDark)
```

`color-scheme` is set automatically (`light` on `:root`, `dark` on `.dark`) so native controls and scrollbars follow the theme.

## Tokens

Seven CSS variables drive the entire theme. Override them anywhere in your app to customize the palette:

```css
.dark {
  --body: #1a1b26;
  --surface: #24283b;
  --surface-2: #1f2335;
  --input: #1f2335;
}
```

## Utilities

Short semantic classes for app shells and custom layout — prefer these over hardcoded slate/white when building theme-aware UI:

```tsx
<div className='bg-body text-fg min-h-dvh'>
  <header className='border-line border-b bg-surface px-4 py-3'>
    <h1 className='font-semibold'>Dashboard</h1>
    <p className='text-muted text-sm'>Welcome back</p>
  </header>
  <main className='p-4'>
    <div className='rounded-lg border border-line bg-surface p-4'>
      <p className='text-fg'>Card content</p>
      <p className='text-muted text-sm'>Supporting detail</p>
    </div>
  </main>
</div>
```

**`html`** and **`body`** already use **`bg-body text-fg`** from preflight — toggling **`.dark`** on `<html>` updates the whole page background.

## Tailwind dark variant

The library registers **`@custom-variant dark (&:is(.dark *))`** in **`core/src/styles/tailwind.css`**. Use **`dark:`** for one-off overrides in app-specific layout:

```tsx
<div className='bg-white dark:bg-surface text-slate-900 dark:text-fg border border-slate-200 dark:border-line'>
  …
</div>
```

Works with responsive and state variants: **`md:dark:bg-surface`**, **`hover:dark:bg-surface`**, etc.
