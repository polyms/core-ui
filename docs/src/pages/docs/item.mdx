Ghost-style interactive utilities for **nav items, list rows, and menu entries**. Each variant is transparent at idle and applies a tinted background + accent text on **`:hover`**, **`:active`**, and the persisted **`.active`** class — the same transparent hover/active pattern used across library components.

## Usage

Apply the variant alongside your own layout classes (padding, radius, flex):

```tsx
const button = (
  <button className='flex w-full items-center gap-2 rounded-lg px-3 py-2 text-sm item-primary'>
    <Icon /> Dashboard
  </button>
)
```

Mark the current row with **`.active`** to persist the selected state:

```tsx
const link = (
  <a className='... item-primary active' aria-current='page'>
    Dashboard
  </a>
)
```

Neutral chrome (sidebars, settings lists) typically uses **`item-light`**, with **`item-danger`** for destructive rows:

```tsx
const nav = (
  <nav className='rounded-xl border border-line bg-body p-2'>
    <a className='... item-primary active'>Dashboard</a>
    <a className='... item-light'>Profile</a>
    <a className='... item-light'>Settings</a>
    <button className='... item-danger'>Delete account</button>
  </nav>
)
```

## Notes

- Variants set **only** color + background; supply your own `display`, `padding`, `border-radius`, and `gap`.
- Hover styles are wrapped in `@media (hover: hover)` so touch devices don't get stuck hover states.
- Tints use `color-mix(in oklab, …)` so they adapt automatically in **dark mode** — no extra classes needed.
- `item-light` / `item-dark` resolve through the **`--muted`**, **`--fg`**, and **`--surface`** semantic tokens, so they invert with the theme.
