Compound accordion built on **[Base UI Accordion](https://base-ui.com/react/components/accordion)**. **`Accordion`** (root), **`Accordion.Item`**, **`Header`**, **`Trigger`**, and **`Panel`** apply bundled classes (**`accordion`**, **`accordion-item`**, …) so motion and layout work once package styles are loaded. Override or extend with **`className`** as usual.

## Usage

### Shorthand title on Item

When **`title`** is set, **`Header`**, **`Trigger`**, and **`Panel`** are created for you; **`children`** are rendered inside the panel (with default padding and body text color).

```tsx
import { Accordion } from '@polyms/core-ui'

<Accordion defaultValue={['billing']}>
  <Accordion.Item title="Shipping" value="shipping">
    Free shipping on orders over $50…
  </Accordion.Item>
  <Accordion.Item title="Billing" value="billing">
    We accept major cards…
  </Accordion.Item>
</Accordion>
```

### Full composition

Use the same tree as Base UI when you need custom trigger content, icons, or non-heading headers.

<CodePreview code={composedCode}>
  <ComposedStory />
</CodePreview>

```tsx
import { Accordion } from '@polyms/core-ui'

<Accordion defaultValue={['notifications']}>
  <Accordion.Item value="notifications">
    <Accordion.Header>
      <Accordion.Trigger>How will you notify me?</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Panel>
      Delivery details…
    </Accordion.Panel>
  </Accordion.Item>
</Accordion>
```

Default **`className`** shells are already merged on each part; add utilities on the **`Accordion`** root (for example max width) or on any subpart for one-off layout.

### Multiple open sections

Pass **`multiple`** on the root so more than one panel can stay open. **`defaultValue`** and controlled **`value`** are arrays of item **`value`** strings.

</CodePreview>

```tsx
import { Accordion } from '@polyms/core-ui'

<Accordion defaultValue={['shipping', 'returns']} multiple>
  <Accordion.Item title="Shipping" value="shipping">…</Accordion.Item>
  <Accordion.Item title="Billing" value="billing">…</Accordion.Item>
  <Accordion.Item title="Returns" value="returns">…</Accordion.Item>
</Accordion>
```

## Types

Export aliases include **`AccordionRootProps`**, **`AccordionItemProps`** (includes **`title`**), **`AccordionHeaderProps`**, **`AccordionTriggerProps`**, **`AccordionPanelProps`**, and a merged **`Accordion`** namespace (for example **`Accordion.Item.Props`**) alongside the value export.

## Motion and CSS

Panels animate with Base UI’s **`--accordion-panel-height`** and **`--accordion-panel-width`** (horizontal **`orientation`**). Closed panels use **`data-starting-style`** / **`data-ending-style`** during transitions. Import the library stylesheet (or Tailwind entry) so **`_accordion.css`** rules apply.

## Controlled vs uncontrolled

Use **`defaultValue`** for uncontrolled mode, or **`value`** + **`onValueChange`** when open sections must follow routing or external state. Set **`multiple`** on the **`Accordion`** root to keep several values in the open set.
