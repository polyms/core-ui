A flexible form field wrapper that combines labels, inputs, descriptions, and validation feedback into a cohesive component. Use it to collect user input such as text, email, passwords, or search queries with proper accessibility and visual hierarchy.

## Usage

```tsx
import { Field } from '@polyms/core'

<Field required>
  <Field.Label>Let me know your email?</Field.Label>
  <Field.Control />
  <Field.Description>We'll never share your email with anyone else.</Field.Description>
</Field>
```

## Disabled and read-only

Pass **`disabled`** or **`readOnly`** on **`Field.Control`**. Disabled inputs block interaction and use reduced opacity; read-only inputs keep the value readable with a muted background and suppress hover border and focus ring styling.

## Debounced `onChange` on `Field.Control`

Pass **`debounce`** (milliseconds) on **`Field.Control`** to delay **`onChange`** until typing pauses—useful for filtering or validation without work on every keystroke. Omit **`debounce`** or use **`0`** for synchronous **`onChange`** (native behavior).

```tsx
<Field>
  <Field.Label>Search</Field.Label>
  <Field.Control debounce={300} onChange={(e) => console.log(e.target.value)} />
</Field>
```

## Floating label

**`Field.Floating`** bundles a floating-label input with optional helper text. Pass **`debounce`** when you want **`onChange`** to fire after a quiet period; omit it for updates on every change. Use **`placeholder`** for the description below the field, not the HTML placeholder (the input uses a blank placeholder so the label can animate).

```tsx
import { Field } from '@polyms/core'

<Field.Floating
  label="Search"
  placeholder="Results update as you type"
  debounce={400}
  onChange={(e) => console.log(e.target.value)}
/>
```
